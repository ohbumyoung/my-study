1. 캡슐화
- 중요한 데이터들을 private 명령어를 사용하여 외부에서 직접 접근하지 못하게 한다.
- 필요에 따라 getter, setter를 이용 

2.상속
- 1번 클래스의 기능을 2번클래스에게도 물려준다.
- 1번 클래스에 나이, 이름을 만들고 2번클래스에게 상속하게 되면 나이, 이름을 다시 만들지 않아도 된다.

3.다형성
- 하나의 타입으로 여러 객체를 처리한다.
- 자식이 가지고 있는 기능은 사용하지 못하게 되며, 사용하기 위해서는 형변환(다운 캐스팅)을 사용해야한다.
- exist

4. 추상화
- interface 클래스명을 사용하여 공통된 메소드를 작성한후 implements 클래스명을 만들어 사용한다.
- 공통된 부분을 설계 및 체크하기 용이하다.
